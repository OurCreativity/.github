## About

**OurCreativity** is an Indonesian-born creative community platform designed as the ultimate home for young creators across all disciplines. Founded with the vision of developing the creative spirit of young Indonesians, we provide a space where talented individuals can connect, collaborate, create, and receive appreciation for their work.

This GitHub organization is maintained by the **Coding Division**, serving as the technical backbone of the entire OurCreativity ecosystem.

### Coding Division

The Coding Division was officially launched in **January 2025** as part of the OurCreativity v5.0 Release (Revolution Edition). We serve as the technical pillar supporting the entire ecosystem.

**What We Do**

## Contributing

We welcome contributions from developers who share our mission of nurturing young Indonesian creators.

### Getting Started

```bash
# 1. Fork the repository
# 2. Clone your fork
git clone https://github.com/YOUR_USERNAME/REPO_NAME.git

# 3. Create a feature branch
git checkout -b feature/your-feature-name

# 4. Make your changes and commit
git commit -m "feat: add your feature description"

# 5. Push and create a pull request
git push origin feature/your-feature-name
```

### Commit Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/) specification.

<div align="center">

| Type | Description | Example |
|:-----|:------------|:--------|
| `feat` | New feature | `feat: add portfolio gallery` |
| `fix` | Bug fix | `fix: resolve login issue` |
| `docs` | Documentation changes | `docs: update API reference` |
| `style` | Code style changes | `style: format with prettier` |
| `refactor` | Code refactoring | `refactor: optimize database queries` |
| `test` | Adding or updating tests | `test: add unit tests for auth` |
| `chore` | Maintenance tasks | `chore: update dependencies` |

</div>

### Guidelines

- Follow the established code style and conventions
- Write meaningful commit messages
- Include documentation for new features
- Ensure all tests pass before submitting
- Be respectful and constructive in discussions

For detailed guidelines, refer to `CONTRIBUTING.md` in each repository.
